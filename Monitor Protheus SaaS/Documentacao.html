Weepulse Monitor

Guia de Implantação e Operação do Sistema de Observabilidade para TOTVS Protheus

🛠️ 1. Instalação Automatizada

O Weepulse Monitor possui um instalador silencioso desenhado para servidores Windows, que configura o ambiente Python e registra o serviço web automaticamente.

Passo a Passo de Implantação

  1. Extraia a pasta do Weepulse no diretório definitivo do servidor do cliente (Ex: C:\Weepulse).
  2. Abra o PowerShell como Administrador.
  3. Execute o comando de permissão: Set-ExecutionPolicy Bypass -Scope Process -Force.
  4. Arraste o arquivo Instalar_Weepulse.ps1 para a tela do PowerShell e pressione Enter.
  5. O sistema fará o download das dependências e criará o Serviço do Windows "WeepulseMonitor".
  6. Acesse o painel pelo navegador em http://localhost:5001.

⚙️ 2. Configurações Iniciais (Menu 7)

A primeira etapa após a instalação é configurar a comunicação do Weepulse com a rede, banco de dados e provedor de e-mail.

  • Rede (WinRM): Informe um usuário com privilégios de Administrador no domínio para que o sistema possa coletar dados remotamente sem precisar instalar agentes nos outros servidores.
  • Banco SQL: Insira as credenciais do SQL Server. Isso habilitará o Menu 3 (Monitoramento DBA).
  • SMTP (Alertas): Configure a conta de envio. Utilize o botão "Testar Envio" para confirmar a comunicação.
  • Limites: Defina a margem crítica para consumo de CPU, Memória e espaço mínimo de Disco Livre.
  • Reinicialização Programada: Agende o horário diário e os dias da semana para a rotina de limpeza de memória da madrugada.
📷 COLE O PRINT AQUI: Tela do Menu 7 (Configurações).

🔄 3. Auto-Healing de Serviços (Menu 1)

Este é o núcleo operacional para o ERP. O sistema classifica automaticamente serviços como APPSERVER, DBACCESS e LICENSE através de inteligência de leitura de caminhos executáveis.

Como funciona o Auto-Healing?

Ao ativar o botão Healing: ON para um serviço, o Weepulse passa a vigiá-lo em tempo real. Se uma queda for detectada (Status STOPPED):

  1. O motor localiza o diretório raiz do executável e lê o arquivo .ini para descobrir o nome real do log.
  2. Extrai as últimas 50 linhas críticas do arquivo de console.
  3. Dispara um comando via PowerShell para forçar o reinício do serviço.
  4. Envia um e-mail probatório contendo o trecho do log que antecedeu a queda.
Reinício Sequencial Seguro: O botão "Reiniciar Todo o Sistema" suspende temporariamente os monitores, desliga os AppServers, desliga os DBs, reinicia o License, aguarda 10 segundos para estabilização, religa os DBs e, por fim, religa os AppServers.

📈 4. Monitoramento de Servidores (Menu 2)

O acompanhamento de hardware do Weepulse é contínuo e não depende de abas abertas no navegador, rodando como uma Background Task silenciosa no servidor.

  • Gráfico de Histórico Contínuo: A CPU e a Memória de todos os servidores cadastrados são amostradas no banco de dados. Ao abrir a tela, o histórico é plotado em tempo real, eliminando "buracos" de medição.
  • Análise de Discos: Traz a volumetria de todas as partições montadas e alerta em vermelho capacidades críticas.
  • Event Viewer Integrado: O sistema traz para a interface web os "Erros Nível 1 e 2 (System/Application)" gerados nas últimas 24 horas pelo Windows, facilitando auditorias rápidas.
📷 COLE O PRINT AQUI: Gráficos de Hardware e Event Viewer.

🗄️ 5. Monitoramento de Banco de Dados (Menu 3)

Uma ferramenta completa de Health Check para DBA. Executa um script nativo otimizado no SQL Server que avalia dezenas de parâmetros críticos e retorna o diagnóstico renderizado.

  • Gargalos Atuais: Exibe "Deadlocks" das últimas horas e "Processos Bloqueados (Travados)" no exato momento do clique, com indicação do usuário e da Query que está travando o banco.
  • Queries Mais Lentas: Mostra as 20 transações mais pesadas executadas recentemente (Top Queries), detalhando tempo médio e I/O gerado.
  • Saúde Estrutural: Análise da Page Life Expectancy (PLE), saturação do TempDB e fragmentação severa de índices (acima de 30%).
  • Sugestões de Índices (Missing Indexes): O SQL analisa quais índices faltam para melhorar a performance (impacto > 70%) e o Weepulse já gera o script CREATE INDEX pronto para copiar e colar.
📷 COLE O PRINT AQUI: Dashboard de Monitoramento do Banco de Dados.

✅ 6. Checagem de Requisitos Oficiais (Menu 4)

Evita problemas de lentidão atestando se a infraestrutura do cliente atende ao documento oficial de dimensionamento da TOTVS para as versões mais recentes.

O avaliador verifica o Role do servidor (ERP ou Banco de Dados) e compara:

  • Cores de CPU e Frequência Base (GHz).
  • Quantidade de Memória RAM necessária vs Encontrada.
  • Tamanho recomendado para discos de SO e Dados.
  • Configurações de rede, Collation do SQL Server e Latência de Ping (interna e externa).
📷 COLE O PRINT AQUI: Tela de avaliação de requisitos (Aprovado/Reprovado).

📄 7. Analisador de Logs do Protheus (Menu 5)

Centraliza a investigação de erros (Access Violations, Fatal Errors) sem precisar de acesso RDP ou FTP às pastas dos servidores.

  • Varredura Automática: O Weepulse busca dinamicamente todos os executáveis do ambiente, ignora arquivos nativos do Windows e lista apenas instâncias ativas do Protheus.
  • Leitura Inteligente: Localiza a chave consolefile= nos arquivos .ini para exibir com precisão o log daquele serviço específico.
  • Highlighter Customizável: O usuário pode cadastrar palavras-chave (ex: "ERROR", "THREAD", "FALHA") no banco de dados e o Weepulse marcará essas linhas com cores vivas para leitura dinâmica.
📷 COLE O PRINT AQUI: Leitor de Logs com as palavras-chave destacadas.

🗺️ 8. Documentação de Ambiente (Menu 6)

Módulo voltado para engenharia e topologia. Faz a engenharia reversa do ambiente do cliente lendo todos os arquivos de configuração (INIs).

  • Desmonta a estrutura de portas (TCP, WebApp, REST, Portal).
  • Exibe o License Server referenciado por cada AppServer.
  • Lista os Envs de conexão com o banco de dados e o Startpath/Sourcepath dos dicionários de dados do Protheus.
  • Bypass de Limites: Usa uma tecnologia de fragmentação de Base64 para conseguir enviar grandes volumes de script via WinRM, evitando a restrição clássica de caracteres da Microsoft.
